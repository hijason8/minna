# 部署到網上一一步一步教學（含概念說明）

這份教學會帶你把「大家的日本語」從「只能在自己電腦跑」變成「在網路上有一個網址，手機出門也能用」。每一步都會先講**為什麼這樣做**，再講**怎麼做**。

---

## 第一部分：先搞懂幾個名詞

### 什麼是「部署」？
- **本機執行**：程式只在你電腦裡跑，關掉電腦或沒連同一個 Wi‑Fi，別人（或你的手機）就連不到。
- **部署到網上**：把程式放到**雲端主機**上跑，主機 24 小時開著，並給你一個 **網址（例如 https://xxx.up.railway.app）**。只要連上網路，任何裝置打開這個網址就能用。

所以「部署」= 把程式搬到雲端、讓它用網址對外提供服務。

---

### 為什麼需要 GitHub？
- 雲端主機（例如 Railway）**不會直接讀你電腦裡的資料夾**，它需要從某個地方**拿到你的程式碼**。
- **GitHub** 是一個「放程式碼的網站」。你把程式碼**上傳（push）**到 GitHub 的某個**倉庫（repository）**，Railway 就可以從那個倉庫**拉程式碼下來**，在雲端幫你安裝套件、啟動程式。
- 簡單記：**GitHub = 程式碼的雲端備份 + 讓 Railway 能抓得到**。

---

### 什麼是 Railway？
- **Railway** 是一種 **PaaS**（Platform as a Service）：你不用自己買主機、裝作業系統，只要把程式碼給它，它幫你**建好環境、裝好依賴、開一個網址**。
- 我們選 Railway 的原因：可以一鍵加 **PostgreSQL 資料庫**，並自動把連線網址（`DATABASE_URL`）給你的程式用，很適合「一步到位上雲端」。

---

### 為什麼要加「PostgreSQL」？
- 你本機現在用的是 **SQLite**（一個檔案 `app_data.db`）。檔案存在你電腦裡，一關機就只在本地。
- 部署到雲端時，主機**重開或重新部署**時，本機檔案可能被清掉，所以不適合把 SQLite 檔案放在雲端當正式資料庫。
- **PostgreSQL** 是「真正的」資料庫服務：資料存在 Railway 的資料庫主機裡，**不會因為你的程式重啟就消失**，而且可以從環境變數 `DATABASE_URL` 連線，程式不用改路徑，只要讀這個變數就好。

---

## 第二部分：事前準備（本機）

### 步驟 1：確認專案能本機跑
在終端機（專案目錄、已啟用虛擬環境）執行：

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

瀏覽器開 `http://127.0.0.1:8000/static/index.html`，能正常看到畫面、操作沒問題再繼續。

**為什麼要先確認？** 本機都跑不起來的話，部署上去也會失敗；先排除本機問題，部署時只要專心處理「雲端環境」。

---

### 步驟 2：安裝 Git（若還沒裝）
到 https://git-scm.com 下載並安裝，或 macOS 用：

```bash
xcode-select --install
```

**什麼是 Git？** 一個「版本控制」工具，會記錄你每次改了哪些檔案。GitHub 是基於 Git 的網站，所以要把程式碼推上去，本機一定要有 Git。

---

### 步驟 3：註冊 GitHub 帳號
1. 打開 https://github.com
2. 點 **Sign up**，用 email 或 Google 註冊一個帳號。

---

### 步驟 4：在 GitHub 建立一個「倉庫」
1. 登入後右上角 **+** → **New repository**
2. **Repository name** 自己取（例如 `minna-nihongo`）
3. 選 **Public**
4. **不要**勾 "Add a README"（我們本地已有專案，要從本地上傳）
5. 點 **Create repository**

記下倉庫網址，例如：`https://github.com/你的帳號/minna-nihongo`

**倉庫（repository）** = 一個專案在 GitHub 上的「資料夾」，裡面會放你 push 上去的程式碼。

---

### 步驟 5：把本機專案推送到 GitHub
在**專案根目錄**（`/Users/jasonchu/Desktop/L`）打開終端機，依序執行：

```bash
# 若從沒在這個資料夾用過 Git，先初始化
git init

# 把目前檔案加入「待提交」
git add .

# 第一次提交（commit = 存一份快照）
git commit -m "Initial commit: 大家的日本語 app"

# 告訴 Git 遠端倉庫在哪（網址換成你剛建的）
git remote add origin https://github.com/你的帳號/你的倉庫名.git

# 主分支叫 main，並推上去
git branch -M main
git push -u origin main
```

若 GitHub 要求登入，可用 **Personal Access Token** 當密碼（GitHub → Settings → Developer settings → Personal access tokens → 產生一個，權限勾選 repo）。

**概念**：
- `git add .`：把目前目錄的變更列入「待提交」。
- `git commit`：在本地做一次「存檔」。
- `git push`：把這個存檔**推送到 GitHub**，雲端就有你的程式碼了。

專案裡已有 **`.gitignore`**，所以 `.venv`、`app_data.db` 等不會被 push 上去，不會把虛擬環境或本機資料庫傳到網路上。

---

## 第三部分：在 Railway 部署

### 步驟 6：註冊 / 登入 Railway
1. 打開 https://railway.app
2. 點 **Login** → 選 **Login with GitHub**，授權 Railway 讀取你的 GitHub（用來拉程式碼）。

---

### 步驟 7：建立新專案並加資料庫
1. 登入後點 **New Project**
2. 畫面會問你要加什麼：
   - 選 **Add PostgreSQL**（或 **Database** → **PostgreSQL**）
3. Railway 會自動建立一個 Postgres，並在專案裡出現一個 **PostgreSQL** 的方塊。

**為什麼先加資料庫？** 因為我們要讓「網站程式」連到這個資料庫；先建好資料庫，之後加「網站程式」時才能把連線網址（`DATABASE_URL`）掛上去。

---

### 步驟 8：把 GitHub 倉庫加進來當「Web Service」
1. 在同一個專案裡，點 **New** 或 **+** 再選 **GitHub Repo**
2. 選你剛 push 的倉庫（例如 `minna-nihongo`）
3. Railway 會偵測到是 **Python** 專案，並用你的 `requirements.txt` 來安裝套件；畫面上會多一個 **Web Service** 方塊（名字通常是你的倉庫名）

**概念**：一個「Service」= 一個會跑的服務。PostgreSQL 是一個 Service，你的 FastAPI 程式是另一個 Service；兩個都要在同一個 Project 裡，才能互相連線。

---

### 步驟 9：把「網站」連到「資料庫」
1. 點進 **Web Service**（你的 FastAPI 那格）
2. 上方或左側找 **Variables**（環境變數）分頁
3. 看是否已經有 **DATABASE_URL**：
   - **若已有**：多半是 Railway 自動從 Postgres 連過來的，不用改。
   - **若沒有**：點 **Add Variable** 或 **New Variable**，選 **Add Reference**（或類似選項），從 **PostgreSQL** 那個服務裡選 **DATABASE_URL**，這樣 Web Service 就會得到資料庫連線網址。

**為什麼要 DATABASE_URL？** 你的程式碼裡用 `get_database_url()` 讀環境變數；Railway 把 Postgres 的連線字串放進 `DATABASE_URL`，程式在雲端啟動時就會自動連到 Postgres，不用改程式。

---

### 步驟 10：確認怎麼啟動（Procfile）
1. 專案根目錄已經有一個 **Procfile**，內容是：
   ```text
   web: uvicorn app.main:app --host 0.0.0.0 --port $PORT
   ```
2. Railway 通常會自動偵測到 Procfile 並用這行來啟動。
3. 若沒自動偵測：在 Web Service → **Settings** → **Deploy** 或 **Build** 區塊，找到 **Start Command**（或 **Custom Start Command**），手動填：
   ```bash
   uvicorn app.main:app --host 0.0.0.0 --port $PORT
   ```
   `$PORT` 是 Railway 給的埠號，不要改成固定數字。

**概念**：`--host 0.0.0.0` 表示「接受從外面連進來的請求」；在雲端一定要這樣設，否則只有主機內部能連。

---

### 步驟 11：等部署完成並產生網址
1. 儲存設定後，Railway 會自動 **Build**（安裝套件）然後 **Deploy**（啟動程式）。
2. 在 **Web Service** 的 **Deployments** 分頁可以看到進度；**Build 成功、Deploy 成功** 後才算完成。
3. 到 **Settings** → **Networking**（或 **Generate Domain**）→ 點 **Generate Domain**。
4. Railway 會給一個網址，例如：`https://minna-nihongo-production-xxxx.up.railway.app`

這個網址就是你的「大家的日本語」在網上的家；任何人（包括你的手機）只要連上網路，打開這個網址就能用。

---

### 步驟 12：第一次打開與建表
1. 在瀏覽器開：`https://你得到的網址` 或 `https://你得到的網址/static/index.html`
2. 第一次請求時，程式會跑 `init_db()`，對 Postgres 執行 `create_all()`，**自動建好所有資料表**，不用你手動下 SQL。
3. 之後就可以在網頁裡用「新增單字／短語、文法、測驗」等功能；資料都會存在 Railway 的 Postgres 裡。

---

## 第四部分：之後怎麼更新程式？

你改過程式碼、想讓網站也更新時：

1. 在本機專案目錄：
   ```bash
   git add .
   git commit -m "說明你做了什麼改動"
   git push
   ```
2. Railway 若已和 GitHub 連動，會自動偵測到 push，**自動重新 build 並部署**；等一兩分鐘再重新整理網址即可。

---

## 第五部分：整理一張「流程圖」

```
本機專案 (L/)
    │
    │  git push
    ▼
GitHub 倉庫 (程式碼在這裡)
    │
    │  Railway 從這裡拉程式碼
    ▼
Railway 專案
    ├── PostgreSQL (資料庫，提供 DATABASE_URL)
    └── Web Service (跑 uvicorn，讀 DATABASE_URL，對外給網址)
            │
            ▼
    https://xxx.up.railway.app  ← 你用手機 / 電腦打開這個網址
```

## 常見問題簡答

| 問題 | 說明 |
|------|------|
| Build 失敗、找不到套件 | 確認 `requirements.txt` 有被 push 上去，且內容包含 `sqlalchemy`、`psycopg2-binary`。 |
| 開網址出現 500 錯誤 | 多半是程式連不到資料庫。到 Web Service 的 **Deployments** → 點最新一次 → 看 **Logs**，檢查有沒有 `DATABASE_URL` 或連線錯誤。 |
| 本機資料會不會上傳？ | 不會。`.gitignore` 已排除 `app_data.db`；部署後用的是 Postgres，資料在雲端，本機 SQLite 仍只在你自己電腦。 |
| 為什麼本地跑的內容和網頁上的不一樣？ | 見下方「本地 vs 網頁的資料庫」說明。 |
| 要付費嗎？ | Railway 有免費額度，個人小用量通常夠用；超過會要求綁卡或升級。 |

---

### 本地 vs 網頁的資料庫（為什麼資料不一樣）

| 環境 | 用的資料庫 | 資料在哪裡 |
|------|------------|------------|
| **本地**（你電腦跑 `uvicorn`） | 預設 **SQLite**，檔案是專案裡的 `app_data.db` | 在你電腦硬碟 |
| **網頁**（Railway 上的網址） | **PostgreSQL**，由 Railway 提供，連線寫在 `DATABASE_URL` | 在 Railway 的雲端主機 |

所以：

- 本地和網頁是**兩份完全獨立的資料**，不會自動同步。
- 你在本地新增的題庫、單字、文法，只存在本地的 `app_data.db`。
- 你在網頁上新增的，只存在 Railway 的 Postgres；你在手機開網址看到的也是這份。

**可以怎麼做？**

1. **接受兩邊分開**：本地當「測試／開發」、網頁當「正式使用」；要讓網頁有資料，就上網站在網頁上匯入或新增。
2. **想讓網頁和本地有一樣的資料**：在網頁版用「新增題庫」等功能重新匯入或建立；或由懂 DB 的人從本地匯出 SQLite 再匯入 Postgres（較進階）。
3. **想讓本地連到網頁的資料庫**（不建議常態使用）：在本地設環境變數 `DATABASE_URL` 指向 Railway 的 Postgres 連線網址，本地程式就會改連雲端 DB；同一份資料，兩邊看到的就一樣，但改壞會影響網頁。

若你卡在某一步，把「步驟編號 + 畫面上的錯誤或截圖」貼出來，我可以依你的狀況一步一步幫你對應。
